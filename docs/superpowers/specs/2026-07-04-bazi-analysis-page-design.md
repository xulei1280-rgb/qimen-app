# 八字分析独立页面设计稿

## 目标

在现有奇门遁甲网页之外新增一个独立的八字分析页面 `bazi.html`。奇门页只提供一个清晰入口跳转，不把八字功能塞进奇门盘面。

第一版定位是“问真式录入体验 + 命格/格局/喜用神核心分析”，不是完整复刻问真专业细盘。

## 已确认需求

- 八字页作为独立页面。
- 奇门页标题下方放一个「八字分析」入口按钮。
- 视觉风格采用浅色问真式：浅灰背景、白色大卡片、金色重点、黑色主按钮、五行彩色干支。
- 第一版只支持公历出生时间。
- 出生时间在表单上显示为单行，例如 `1990年05月07日 07:00`，点击后展开或弹出年月日时分选择。
- 出生地址使用常用地点 + 搜索匹配，不做问真式省市区三级选择。
- 默认常用地点：
  - 云南昆明
  - 曲靖麒麟区
  - 师宗县
  - 罗平县
  - 泸西县
  - 陆良县
- 选中出生地后自动带出经纬度，用于真太阳时校准。
- 真太阳时默认勾选，用户可以取消。
- 八字 AI 保留同一张八字盘的上下文，但不限制只能问同一件事。用户可以围绕同一八字连续问命格、事业、财运、感情、流年等不同主题。
- 保存记录需要保存整张八字盘、本地计算结果和 AI 对话历史。

## 非目标

第一版暂不做：

- 农历输入。
- 手动输入四柱。
- 完整大运/流年长表。
- 问真专业细盘的全部神煞和细项复刻。
- 多流派格局硬编码定论。

这些可以作为后续阶段扩展。

## 页面结构

### 奇门页入口

在 `index.html` 的主标题和副标题下方增加一个轻量入口：

- 文案：`八字分析`
- 行为：跳转到 `bazi.html`
- 样式：与现有奇门页风格协调，不抢起盘控件。

### 八字录入页首屏

使用居中大白卡结构，模仿问真录入页的信息顺序：

1. 命主姓名
2. 性别
3. 出生时间
4. 出生地址
5. 真太阳时选项
6. 校正信息预览
7. 开始排盘按钮

出生时间行：

- 默认只显示格式化后的时间。
- 点击后展开选择面板。
- 选择面板包含年、月、日、时、分。
- 修改后自动收起或点击确认收起。

出生地址行：

- 显示已选地点名称。
- 点击后展开搜索/选择面板。
- 面板顶部提供搜索框。
- 下方显示常用地点。
- 输入关键词时按名称匹配。
- 选择地点后更新地点名、经纬度和真太阳时预览。

真太阳时预览：

- 勾选时显示：`真太阳时：1990-05-07 05:58` 或近似值。
- 同时显示地点经纬度，例如：`北纬25.4951 东经103.805`。
- 未勾选时显示：`未启用真太阳时，将按输入时间排盘`。

### 八字结果页

结果区采用白底专业表格与摘要卡结合。

第一屏结果：

- 日主属性卡：如 `壬水`。
- 四柱摘要卡：如 `庚午 辛巳 壬申 癸卯`。
- 旺衰参考卡：如 `偏强 / 偏弱 / 中和`。
- 喜用参考卡：显示喜用五行。

四柱表：

- 列：年柱、月柱、日柱、时柱。
- 行：
  - 主星
  - 天干
  - 地支
  - 藏干
  - 副星
  - 星运/十二长生
  - 空亡
  - 纳音

结构分析区：

- 五行强弱条。
- 日主旺衰说明。
- 格局参考。
- 喜用神/忌神参考。
- 干支关系提示。

AI 分析区：

- 输入框：用户可以问命格、事业、财运、感情、流年等。
- 输出方式：
  - 简断
  - 详断
  - 格局层次
  - 大运流年思路
  - 只看结论
- AI 上下文绑定当前八字盘。
- 换人或重新排盘时开启新的对话上下文。

保存记录区：

- 保存当前八字盘。
- 保存 AI 对话历史。
- 加载记录时恢复八字盘和 AI 上下文。

## 数据与计算

### 地点数据

第一版内置常用地点数组，每项包含：

- 展示名
- 搜索别名
- 经度
- 纬度

初始地点：

| 地点 | 用途 |
| --- | --- |
| 云南昆明 | 常用省会 |
| 曲靖麒麟区 | 用户样本地点 |
| 师宗县 | 用户常用地点 |
| 罗平县 | 用户常用地点 |
| 泸西县 | 用户常用地点 |
| 陆良县 | 用户常用地点 |

经纬度以公开地理坐标的近似值内置。真太阳时只依赖经度；纬度主要用于展示。

### 真太阳时

复用现有奇门页的真太阳时思路：

- 经度校正：`(经度 - 120) * 4` 分钟。
- 加上均时差近似值。
- 得到起盘用时。

校验样本：

- 问真链接样本：`1990-05-07 07:00`，曲靖麒麟区。
- 问真真太阳时：`1990-05-07 05:58`。
- 本地计算允许 1-2 分钟近似误差，但不能跨时辰。
- 四柱应为：`庚午 辛巳 壬申 癸卯`。

### 四柱

第一版复用现有 `qimen.js` 的基础干支函数：

- `getYearGZ`
- `getMonthGZ`
- `getDayGZ`
- `getHourGZ`

日柱规则沿用奇门页已验证逻辑：

- 23 点后按子初换日。

风险：

- 现有节气采用近似处理。节气交接当天需要在页面显示提醒，避免误判月柱。

### 八字分析本地参考

本地只给“参考线索”，不硬下最终流派定论：

- 十神。
- 藏干。
- 副星。
- 十二长生。
- 空亡。
- 纳音。
- 五行分布。
- 日主旺衰粗判。
- 月令取格参考。
- 喜用神参考。

格局、层次、喜用神的最终综合判断交给 AI，并在 prompt 中要求说明依据和不确定性。

## AI Prompt 原则

八字 AI 与奇门 AI 分开：

- 不使用“一个盘只能问同一件事”的限制。
- 保留当前八字盘的聊天上下文。
- 不同主题可以连续追问。
- 重新排盘后重置上下文。

AI 输入应包含：

- 出生时间和是否启用真太阳时。
- 四柱。
- 日主五行。
- 月令。
- 十神/藏干概要。
- 五行统计。
- 旺衰参考。
- 格局参考。
- 喜用参考。
- 用户问题。
- 历史对话。

AI 输出要求：

- 不恐吓。
- 不绝对化。
- 对不同流派可能有分歧的地方说明“参考”。
- 优先回答用户当前问题。
- 问命格层次时重点解释格局、成败、用神、忌神和现实取向。

## UX 细节

- 初次打开八字页默认载入问真样本作为示范，避免空页面让人不知道如何填写。用户修改任一字段后按自己的资料排盘。后续可增加“清空”和“当前时间”快捷按钮。
- 手机端第一屏应先看到录入卡，不要直接露出开发式下拉框。
- 表单字段必须有清晰标签，避免“不知道填什么”。
- 选择地点后立即刷新真太阳时预览。
- 取消真太阳时后，排盘摘要明确显示“按输入时间排盘”。
- 保存记录只保存在当前浏览器本地。

## 验证

必须验证：

- `node .\check-baselines.js` 仍通过，确保奇门逻辑未受影响。
- `index.html` 和 `bazi.html` 内联脚本解析通过。
- 问真样本输出四柱为 `庚午 辛巳 壬申 癸卯`。
- 手机本地预览可打开 `bazi.html`。
- 奇门页入口能跳转到八字页。
- 八字页返回入口能回到奇门页。

- 加常用地点搜索。
- 加真太阳时预览。
- 加核心结果区。
- 加 AI 上下文与保存。
- 保持奇门页只新增入口。
