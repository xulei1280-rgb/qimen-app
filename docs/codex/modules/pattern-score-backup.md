## 当前原始分公式

主函数：`bazi-engine.js` 的 `patternStructureRawScore`。

| 计分项 | 当前权重 |
| --- | --- |
| 基础分 | `50` |
| 特殊格局严格成立 | `+12` |
| 严格成格组合 | `+9` |
| 普通格局已成 | `+6` |
| 成而有瑕 | `+1` |
| 格局已破 | `-9` |
| 格局待成 | `-5` |
| 成格完整度 | `round((满足比例 - 0.5) * 12)` |
| 作用链全部可达 | `+8` |
| 作用链部分可达 | `+2` |
| 作用链全部不可达 | `-6` |
| 清 / 较清 / 可清 / 有瑕 | `+8 / +6 / +3 / 0` |
| 冲战 / 不清或偏浊 / 浊 / 待清 / 待辨 | `-4 / -5 / -8 / -3 / -2` |
| 病点已解 | 每项 `+2`，合计最多 `+4` |
| 病点部分化解 | 每项 `-2` |
| 病点未解 | 每项 `-6` |
| 气势病象轻度 / 明显 / 严重 | 每项 `-2 / -4 / -7` |
| 清秀气象 | 每项 `+2`，合计最多 `+4` |

最后将原始分四舍五入并限制在 `0-100`。理论百分位使用同分中位秩；
百分位先生成候选档，再由 `finalizePatternLevel` 的子平硬门槛决定上下限和最终五档。

## 当前去重状态

- 已实现：`patternDiseaseAssessment` 按病点 `family` 合并同源问题；同一承载不足、
  官伤冲战等不会因为多个格名重复生成病点。
- 已实现：已完全化解的病点不再通过硬门槛重复压档；部分化解和未解只按残病限制上限。
- 已实现：气势病象在最终五档中按程度设置上限，不因名称直接判定最终档位。
- 尚未实现：原始结构分各维度之间没有统一的因果键；格局状态、完整度、作用链、
  清浊、病药和气势病象仍可能描述同一上游原因。后续只能在审核确认为同源时去重，
  不能把本来独立的结构证据一律合并。

## 回滚规则

1. `d95c0fd3faf7365cee82c339c5129194848121c2` 是当前评分代码的精确恢复来源。
2. 后续 `PAT-208` 修改必须使用独立、聚焦的提交；回滚时优先逐个撤销这些评分提交，

   未确认的实验版本不得覆盖此备份。
