# 格局喜用确认偏误踩坑记录

## 事件结论

2026-07-14 复核发现，开发讨论中出现过的真人判断被写成了
`check-bazi-page.js` 的预期答案：已知某盘被讨论为“食神制杀”，测试便固定要求
该盘输出食神制杀、固定病药数量、固定层次，并把同类火直接解释为朋友分财；另一个
真人盘也被固定为土厚埋金、杀印相生和特定层次。规则随后为通过这些测试而放宽，形成
“先知道答案，再补条件”的循环验证。

这不是运行时读取聊天内容，而是开发阶段的确认偏误和数据泄漏。真人反馈只能提出待核
假设，不能成为格局、喜用、层次、行为或分数的真值标签。

## 直接成因

1. 测试按已知人物结论断言最终格名和层次，没有先证明规则条件。
2. 食神制杀曾允许“食神或七杀至少一方透出”，使藏支食神也可被提升为成格。
3. 食伤分工逻辑专门接受“伤官当令、食神不透但藏支制杀”，绕过了食神制杀本身的严格闸门。
4. 条件喜用把同类比劫一律扩写为朋友分财，行为结论没有实际比劫夺财作用链。
5. 评分在错误格名之后给食神制杀高潜力，放大了前置误判。

## 永久禁则

1. 用户、师傅、名人资料、朋友经历和产品反馈只能登记为“待复核假设”，不得直接成为测试期望。
2. 真人盘不得断言“必须是某格、必须是某层次、必须取某用神、必须发生某行为”作为合格条件。
3. 新规则必须按“原典原则 -> 结构化成立条件 -> 最小成立盘 -> 单条件反例 -> 组合反例”顺序落地。
4. 成格判定必须先于评分；评分只能评价已经验真的主格，不能反向救活未成立的格局。
5. 线索、兼见、藏支药神和运中触发不得自动升级为原局成格。
6. 行为出口必须有独立作用链，使用概率化表达；不能从单个喜忌五行直接推出朋友、财富、懒惰或职业事件。
7. 已知真人盘若因本事故保留，只能作为“防止旧错误复发”的匿名反例，断言必须落在可核条件上，不能锁定有利结果或具体档位。
8. 气势病象同样遵守上述边界：不能因为某个真人盘被反馈为“土厚埋金”等术语，就固定要求该盘命中；应先建立可复算的术语规则族，再用理论成立盘、近似盘和反证盘验收，并让所有命盘共用同一入口。
9. 理论基准只使用固定时间网格；任何规则变化都升级规则、引擎和基准版本并重新生成直方图。

## 食神制杀当前验真口径

依据本项目收录的子平主线，食神制杀至少要求：食神与七杀俱有根气；双方透出或得月令；
日主能承泄任杀；食神与七杀作用可达；官杀不混；没有财星承食生杀而使食神贪生忘克。
条件不足只输出“食神制杀线索参考”，并列出未满足项。

本次事故盘为伤官月令，食神仅藏支，正官、七杀同透且日主偏弱。因此只能判为：
伤官格主线；官杀混杂、伤官见官为病；藏支食神有制杀线索或局部药意，但不按食神制杀成格。

## 强制审查清单

- 测试名称是否来自真人身份或已知经历。
- 断言验证的是成立条件，还是在锁定想要的答案。
- 每个成格正例是否至少有一个只差单项的反例。
- 藏支、透干、得令、根气、承载和作用距离是否分别核验。
- 官杀混杂、财转生杀、枭夺食等直接反例是否进入闸门。
- 行为文案是否存在没有作用链的具体人物、财富或职业推断。
- 格名变化后是否重建理论基准，而不是沿用旧百分位。

## 本次修复范围

- 收紧食神制杀和食伤分工成立条件。
- 官杀同透不再只限正官月令才识别混杂。
- 条件比劫不再自动解释为朋友分财。
- 删除真人盘的有利格名、固定层次和固定行为真值测试，改为规则反例与性质测试。
- 重建理论基准并升级版本。
