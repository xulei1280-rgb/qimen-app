# 子平规则交叉索引

本索引用于把原典内容映射到格局喜用模块。它不是把各书混成一套“统一古法”，而是明确默认主线、校正层和历史补充层。

## 项目采用层级

### A. 默认月令格局主线

- 《子平真诠》：月令取用、八格、用神变化、成败救应、相神与格局高低。
- 《三命通会》：以日为主、以月令提纲，补充财官印食杀伤的成立、破损、制化和行运条件。
- 《渊海子平》：交叉核对日主、月令、十神及早期格局术语。

三书意见不一致时，不以书名数量投票。先比较是否同一版本语境、是否讨论同一格局阶段，再以“月令立格、作用链可达、成败救应可核”的默认规则裁决并留下异说。

### B. 结构诊断与校正层

- 《滴天髓阐微》：旺衰、源流、清浊、通关、真假、寒暖燥湿和众寡。
- 《穷通宝鉴》：十干十二月的调候提示。
- 《神峰通考》/《命理正宗》：病药、旺弱损益、格局正说与谬说。

校正层可以说明病点、药神、气势和取用先后，但不能把未满足月令成格条件的组合直接升级为已成贵格。调候只有在寒暖燥湿明确时先处理，且不能反伤格局用神。

### C. 历史与异说层

- 《五行精纪》：保存年命、纳音、禄命、神煞和术语源流。

此层不进入默认格局层次和喜用评分。若未来实现，必须使用独立规则档案与独立输出，不得混入 `ZP-*` 默认规则。

## 默认推理顺序

1. 以日主为主，辨月令司令与月支藏干，先确认是否具备常格立格条件。
2. 核对透干、通根、得势、作用可达与清纯度，区分“已成”“待成”“有瑕”“破格”。
3. 找出成格所依的相神、直接破点与有效救应；名称好听不能替代条件完整。
4. 再用旺衰判断日主是否能任财官、泄秀、制杀或受生，旺衰服务于结构承载，不单独冒充格局。
5. 再看清浊、源流、病药、通关和五行气象，说明为什么可用、受阻或需要去留。
6. 寒暖燥湿明显时加入调候校正；调候不明显时不抢占格局或扶抑优先级。
7. 从格、化格、专旺等特殊格局必须满足完整条件；条件不足只输出线索和未满足项。
8. 大运只改变当前成败与发挥，不倒写原局层次；当前阶段不使用任何真实人物轨迹验证规则。

## 2026-07-14 古籍潜力与特殊格验真落点

当前评分先验真主格，再给且只给一份“古籍格局潜力”。格名文字本身不直接得分，兼容组合与次格也不叠加潜力。潜力只表达结构上限；成格完成度、作用链、清浊、病药和气势仍在各自维度独立结算。

### 六维权重与潜力层级

| 维度 | 权重 | 当前边界 |
| --- | ---: | --- |
| 古籍格局潜力 | 20% | 仅稳定规则 ID 验真的主格取得一份；`S=100`、`A=85`、已成或有瑕常格 `B=70`、待成常格 `50`、破格或未验真 `0` |
| 成格完成度 | 25% | 按主格状态与成立检查完成度结算；特殊格使用自身 `qualification.checks` |
| 作用链 | 20% | 按有效且可达的作用步骤结算；特殊格使用自身 `flowChecks` |
| 清浊去留 | 15% | 结算清纯、阻隔与去留；病药拥有的同一负面原因不在本维重复扣分 |
| 病药净结算 | 15% | 按已解、部分、未解病点结算；同病只结算一次 |
| 气势校正 | 5% | 只作元素病象或清秀气象的小幅校正，不反客为主 |

严格贵格的潜力已经进入原始分，不再额外附加 `偏高` floor；`顶级` 仍须同时通过 P95 候选、成格完整、作用链可达、清纯且无残病或元素病象等硬门槛。若贵格未严格成立，例如伤官配印遇财坏印而只成有瑕，则不保留贵格潜力，退回对应月令常格层级。

## 规则入库模板

每条新增规则至少记录：

```text
ruleId:
sourceSystem: 主流子平 / 调候校正 / 病药诊断 / 历史异说
source: 书名 / 卷篇 / 修订号或扫描页
claim: 古籍条件的现代释义
requiredEvidence: 必须同时满足的命盘证据
counterEvidence: 不成立或降级的反证
result: 已成 / 待成 / 有瑕 / 破格 / 线索
engineeringThreshold: 项目数值门槛及其保守性说明
tests: 成立盘 / 边界盘 / 反例盘
```

古籍未给数值时，`engineeringThreshold` 必须标为项目内部保守门槛；它只能被专项反例测试调整，不能借古籍名义包装成固定真值。
