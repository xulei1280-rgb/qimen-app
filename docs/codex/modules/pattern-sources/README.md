# 格局喜用古籍参考库

本目录保存格局喜用模块采用的原典快照、版本记录和项目规则索引。目标是让每条规则可以回到具体书目与篇章核对，而不是把“古籍有云”当作无来源结论。

## 阅读入口

- [`source-manifest.md`](source-manifest.md)：书目、版本、完整度、许可和本地文件。
- [`rule-crosswalk.md`](rule-crosswalk.md)：各书在本项目中的采用范围、冲突处理和规则主题索引。
- [`checksums.sha256`](checksums.sha256)：全部原始文本与扫描的 SHA-256 完整性校验。
- `texts/`：维基文库原始 Wikitext 快照，不作现代标点或内容改写。
- `scans/`：维基共享资源公版扫描，用于核页、校异和补足电子文本。

## 使用边界

- 原典入库不等于规则自动生效。每条程序规则仍须写明成格条件、反证、来源体系和内部阈值。
- 古籍文字条件与项目数值门槛分开。比例、分数和置信度是工程化实现，不冒充古籍原文。
- 原文、原注、后人评注分层引用。《滴天髓阐微》等评注本不得把评注直接署为原文。
- 同名异书、异名同书和版本差异先记录再裁决，不用多数文本简单投票。
- 默认体系为主流子平月令格局。纳音、神煞、禄命法和单一调候口诀只能进入独立补充层，不能静默改写默认结论。
- 当前验证仅使用可复算的理论命盘网格，不使用名人、朋友、用户或任何真实人物的人生轨迹。
