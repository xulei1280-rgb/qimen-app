## 使用规则

后续继续优化时按以下流程维护：

1. 先用稳定 ID 选中条目，例如 `PAT-101`。
2. 开始编码前把状态改为“进行中”。
3. 每项修改必须增加 `check-bazi-page.js` 对应测试。
4. 完成后记录实现函数、判断依据、测试和仍存不确定性。
5. 不删除已经完成或不采用的条目，保留历史状态。

状态说明：

| 状态 | 含义 |
| --- | --- |
| 已完成（工作树） | 当前格局喜用工作树已经实现并验证 |
| 已确认待做 | 方案已经确认，但尚未编码 |
| 待讨论 | 方向存在，规则或边界仍需商定 |
| 进行中 | 当前正在实现 |
| 暂缓 | 已决定后做，不能顺手扩大范围 |
| 不采用 | 讨论后明确不实施，并保留原因 |

## 已确认待做

### PAT-101 五级理论组合分布

- 状态：已完成（工作树）
- 目标：将“偏低 / 中等 / 偏高 / 高 / 顶级”映射到冻结理论基准库的百分位。
- 暂定分布：`5% / 15% / 60% / 15% / 5%`。
- 百分位边界：
  - 偏低：`P0-P5`
  - 中等：`P5-P20`
  - 偏高：`P20-P80`
  - 高：`P80-P95`
  - 顶级：`P95-P100`
- 目的：高与顶级的候选比例合计20%，其余候选层次合计80%；两端稀少，偏高为主体。
- 验收条件：不能用单盘原始分直接套比例，必须先建立并冻结基准样本。
- 完成日期：2026-07-13。
- 实现结果：理论百分位只生成候选档位；完整网格最终占比为 `4.80% / 15.69% / 60.76% / 16.89% / 1.85%`，不冒充现实人口分布。

### PAT-102 五级硬门槛

- 状态：已完成（工作树）
- 目标：百分位只给候选层次，子平成格条件决定最终上限。
- 暂定门槛：
  - 顶级：候选百分位达到P95，同时严格成格、有力、有情、清纯、无残病和气势病象。
  - 高：至少明确成格，或成而有瑕但救应有效；未通过顶级全部条件时封顶高。
  - 偏高：普通成格、待成但结构流通，或成格有明显瑕疵。
  - 中等：主要条件不足，或破格后只有有限救应。
  - 偏低：理论位置靠后且核心作用链受损、无有效救应。
- 约束：未见明确成格原则上封顶偏高；不能因排名靠前或格名贵重强行判高、顶级。
- 依赖：`PAT-101`、`PAT-103`。
- 完成日期：2026-07-13。
- 实现结果：严格成格组合取得偏高下限但不直接顶档；病药按已解、部分化解、未解净结算，已解病点不重复压档。

### PAT-103 冻结理论基准库

- 状态：理论基准已完成（工作树）
- 目标：为百分位提供稳定比较人群。
- 方案：
  - 从固定年份范围均匀抽取合法出生时间。
  - 使用统一时区、地点和真太阳时规则。
  - 对重复四柱设置明确的去重或人口权重策略。
  - 基准库带版本号，规则升级后不得静默更换样本。
- 输出：每个原局得到稳定的 `rawScore`、`percentile`、`baselineVersion`。
- 已确认边界（2026-07-13）：基准库不拿现实人物的人生轨迹训练，也不把“命好坏”当作已知标签。首期采用可复算的理论出生时间网格，只比较同一套规则下命盘结构分数的相对位置；它解决“分布稳定和可复算”，不证明命理对现实的解释力。
- 当前验证边界（2026-07-13）：只使用可复算的理论命盘网格；不使用名人、朋友、用户或任何真实人物的人生轨迹作验证、校准或阈值依据。若以后改变边界，必须由用户重新明确确认并另开验证方案。
- 完成日期：2026-07-13
- 基准版本：`ZP-TB-1984-2044-D11-H12-v2`。
- 固定口径：1984-02-04 至 2044-02-04，每 11 天、十二时辰中心点、东八区、东经 120°、关闭真太阳时修正、完整四柱去重。
- 复算结果：23,916 个理论时点，23,916 个唯一四柱；冻结直方图哈希见 [`pattern-theoretical-baseline.md`](pattern-theoretical-baseline.md)。
- 输出：每盘返回 `patternRawScore`、`patternPercentile`、`baselineVersion` 和可解释分项；不含真人结果标签。
- 边界：当前只采用理论结构基准，不冒充真实人口分布；人口基准仍不采用。

### PAT-113 百分位同分处理

- 状态：已完成（工作树）
- 必要性：格局评分是离散的，P80、P95 边界可能出现大量同分八字。
- 原则：同一原始分和同一硬门槛结果必须得到同一层次，不能为了强行凑足 5% 或 20% 随机拆分。
- 已确认裁决顺序：先过层次硬门槛；同分先比较主结构质量，再比较成格完整度、作用链、清浊、破格和救应，最后才比较原始分与百分位。
  - 特殊格局严格成立。
  - 严格成格组合：杀印相生、食神制杀、伤官配印、官印相生、财官印相生；必须成格完整、作用可达、清纯可用，且无直接破点。
  - 普通常格已成。
  - 待成、成而有瑕或受损结构。
- 约束：上述组合不按名称彼此硬排高低；同层仍比较成格完整度、作用链、清浊、破格和救应。严格成格组合只取得“原局不低于偏高”的下限与同分优先权，不自动升入高档或顶级。
- 建议：理论比例是目标分布，不是必须精确凑数；边界遇同分和硬门槛回落时允许实际占比偏离。
- 当前输出：`patternLevelTieBreak`（优先层、依据、逐项比较键）与 `patternLevelHardGate`（候选档、上下限、最终档、同分政策）；同分统一采用中位秩百分位，不随机拆分，实际占比允许偏离目标。

### PAT-131 术语作用对象

- 状态：进行中（现有气势病象已统一限定作用对象）

### PAT-143 我生者太过与泄身反伤病象

- 状态：已完成（工作树）
- 必要性：当前 `PAT-123` 主要收录“生我者太过、反而壅滞受生者”的病象，尚缺“我所生者过旺、反而耗伤生者”的另一组常用气势病象。
- 经典主线：《三命通会·元理赋》所列五行生克制化宜忌：
  - 金能生水，水多金沉。
  - 水能生木，木盛水缩。
  - 木能生火，火多木焚。
  - 火能生土，土多火晦。
  - 土能生金，金多土变。
- 术语归类：`我生者太过 / 食伤泄身太过 / 子旺耗母型气势病象`；它们不是正式格局名称。
- 与 `PAT-123` 的反向配对：
  - 土多金埋：土过重使金难受生；水多金沉：金生水过度而金被泄沉。
  - 水多木漂：水过重使木漂浮；木盛水缩：水生木过度而水被耗缩。
  - 木多火塞：木过重使火难明；火多木焚：木生火过度而木被焚耗。
  - 火多土焦：火过重使土焦燥；土多火晦：火生土过度而火被泄晦。
  - 金多水浊：金过重使水不清；金多土变：土生金过度而土气被泄变。
- `土多火晦`重点边界：
  - 火弱、少根或根受损，土得令、透干或成势，才可定为病象。
  - 辰丑湿土偏于敛火、晦光；未戌燥土偏于泄火、燥滞，须分湿燥而不能只数土。
  - 火本身强旺且土能正常泄秀时，不得判为火晦。
  - 木能生火兼疏土，但是否可用仍须校验木的根透和日主承载；水的作用须结合寒暖燥湿，不能机械列为用神。
- 五组共同门槛：
  - 先确认受损五行的日主或十神作用对象。
  - 核验月令、相对气势、透干、根气、位置和实际泄化链。
  - 区分“正常泄秀”“泄身太过”“从势顺泄”三种结果。
  - 有强根、比劫帮身、印星有效承接或特殊格局顺势时，必须降级或判不成立。
- 输出要求：沿用 `PAT-126` 至 `PAT-142` 的统一结构，显示专业词、定义、本盘依据、反证、严重度、行为倾向、作用对象、改善条件、喜用修正和规则来源。
- 行为边界：只能从日主状态、十神职责与现实领域翻译为概率倾向；不得把“土多火晦”固定等同于迟钝、懒散或某种健康问题。
- 验收要求：每个词分别建立成立盘、轻度盘、反向配对盘、正常泄秀盘和特殊格局排除盘；尤其必须成对验证“土多火晦”与“火多土焦”。
- 来源记录：主词采用《三命通会·元理赋》版本；`金多土虚/金多土弱`等异名按 `PAT-138` 记录，不与主词并列重复展示。
- 完成日期：2026-07-12
- 实现位置：`PHENOMENON_RULES`、`elementPhenomenaAnalysis`、`usefulElements`、`applyPhenomenaToStructures`、`applyPhenomenaToClarity`、`remedyAdvice`。
- 已实现词条：水多金沉、木盛水缩、火多木焚、土多火晦、金多土变。
- 判定边界：日主必须是受损五行；食伤一方须达到相对气势门槛，日主无可用强根且承载不足；真从儿格直接排除，强根正常泄秀不作病象。
- 喜用联动：优先取既能生扶日主、又能制约过旺食伤的印星，再配日主比劫恢复承载；过旺食伤列为不宜再增。
- 页面与 AI：复用统一“气势病象”出口，显示术语、作用对象、比例证据、反证、行为倾向、改善条件、来源和规则编号。
- 测试案例：五个成立盘、强根正常泄秀盘、真从儿格排除盘、火土作用对象反例；`check-bazi-page.js`通过，18 个基准盘通过。
