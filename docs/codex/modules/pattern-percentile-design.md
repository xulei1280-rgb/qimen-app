# 格局层次结构分设计

> 统一进度入口：[`docs/codex/OPTIMIZATION-STATUS.md`](../OPTIMIZATION-STATUS.md)

> 版本：`PAT-LV-DESIGN-v1`
> 状态：正式设计，2026-07-15确认采用结构分直接分档
> 模块：完整格局喜用
> 适用文件：`bazi-engine.js`、`bazi.html`、`check-bazi-page.js`
> 数值状态：现行六维权重、维度锚点、结构分边界和高档资格已确认；理论百分位仅作审计
> 禁止：不得根据用户、朋友、名人或其他真人经历倒推格名、分数、P值、权重或档位

## 一、设计目标

本方案评价**原局结构可用性**：原局是否形成可运行的主线，日主能否承载，作用链是否可达，
清浊能否取舍，病点是否获得有效救应，以及整体气势是否形成明确限制。

系统内部依次完成原始结构分、结构分候选档和高档资格裁决；原局对外只形成一个
综合格局层次结论。当前大运另行复核，不改写原局结论。

### 非目标

- 不评价人的价值、道德、幸福或社会身份。
- 不把财富、学历、家庭、健康和现实成就压成一个命运排名。
- 不把理论组合分布表述为现实人口分布。
- 不让当前大运、流年或现实结果改写原局结构分和原局档位。
- 不凭格名、用户描述或单张代表盘直接决定高低或修改通用规则。

## 二、理论优先级

评分顺序服从主流子平判断：

1. 先按月令及完整资格条件确定主格；条件不足只保留线索。
2. 再核成败、相神、反证和救应，判断结构是否真正可用。
3. 再核根气、承载和生克制化先后，判断作用是否有力且可达。
4. 再看清浊去留与病药净结算，已解病点不得重复压制。
5. 寒暖燥湿或明显气势病象只在实际成立时校正，不替代格局主线。

古籍没有提供六维数字比例。本文权重是对上述优先级的工程映射，不冒充古籍原文。
理论交叉依据统一维护在 `pattern-sources/rule-crosswalk.md`。

## 三、结果模型

```text
四柱规则分析
  -> 六维原始结构分
  -> 结构分五档候选
  -> 高档与顶级资格裁决
  -> 原局最终档位
  -> 当前大运第五柱复核（另算）
  -> 普通版、专业版与AI读取同一分析对象
```

| 名称 | 含义 |
| --- | --- |
| 原始结构分 | 六维工程评分，限制在0至100；理论总体不必覆盖0和100端点 |
| 理论百分位 | 原始分在理论合法四柱总体中的诊断位置，只用于审计 |
| 同分区间 | 审计离散评分时使用，不参与单盘定档 |
| 候选档位 | 由未舍入结构分保留一位小数直接映射出的初始五档 |
| 原局最终档位 | 候选档位经过高档与顶级资格裁决后的唯一原局结论 |
| 当前发挥档位 | 当前大运作为第五柱复核后的阶段结果，不改变原局档位 |

## 四、六维原始结构分

### 审计候选权重

```text
raw = round(
  potential * 0.10 +
  formation * 0.25 +
  flow * 0.25 +
  clarity * 0.15 +
  remedy * 0.20 +
  balance * 0.05
)
```

| 维度 | 候选权重 | 唯一评分职责 |
| --- | ---: | --- |
| 格局潜力 `potential` | 10% | 已验真主格在主流子平结构中的理论上限，不按格名文字自动加分 |
| 成格完成度 `formation` | 25% | 主格成立条件、反证与破格条件的实际完成程度 |
| 作用链 `flow` | 25% | 根气、承载、相神及生克制化步骤是否有力、可达且次序成立 |
| 清浊去留 `clarity` | 15% | 主次是否清楚，混杂、冲战能否完成取舍 |
| 病药净结算 `remedy` | 20% | 主要病点被处理后的残余状态，不奖励病本身 |
| 气势校正 `balance` | 5% | 未被其他维度认领的明显偏枯、气候或气势病象校正 |

候选权重只用于全集审计，不是冻结值。各维度的内部数值锚点必须单独版本化，并随全集报告
一起审查；审计前不得把当前实现中的临时锚点写成正式标准。

### 格局潜力进入条件

1. 只读取结构化结果中唯一主格。
2. 主格必须具有稳定规则ID、来源层、成立检查和反证检查。
3. 兼格、线索、显示名称和藏而无力的药意不叠加潜力。
4. 直接破格点成立时，组合格潜力退回其仍然成立的月令常格层级。
5. 普通常格稳定可用时保留常格潜力，不能因没有贵格名称而归零。
6. 特殊格只使用自身资格、顺势链和清浊语境；被其吸收的普通病点不得再次计分。

### 病药规则

- 病本身不加分。
- 药神必须对症、有力、可达，并实际处理对应病点。
- 病点完全处理后不再压档；部分处理只保留残病；未处理按实际严重度限制。
- 无主要病点与主要病点全部处理采用相同的中性偏上语义，不制造额外奖励。
- 同一个药只能处理其真实对应的病点，不能跨病点重复使用。

### 跨维度去重

每个数值组件必须携带：

```text
causeId       上游原因稳定标识
numericOwner  唯一数值归属维度
ruleId        触发规则
evidence      盘面证据
share         组件在所属维度内的占比
```

1. 同一 `causeId` 只能在一个维度产生非零数值影响。
2. 其他维度可以解释同一原因，但必须保持数值中性并指向 `numericOwner`。
3. 格局潜力表达理论上限，成格完成度表达当前完成程度，不得复制同一个状态分。
4. 清浊问题若由病药家族认领，清浊维度不得再次扣分。
5. 已处理的病点不得在硬门槛中按原病再次压档。
6. 气势校正不得重算已由承载、清浊或病药拥有的原因。

## 五、理论合法四柱总体

理论总体固定为518,400种等权合法四柱结构：

```text
60种年柱 × 12种月支 × 60种日柱 × 12种时支 = 518,400种
```

- 月干按年干与月支通过五虎遁唯一确定。
- 时干按日干与时支通过五鼠遁唯一确定。
- 月支按寅至丑十二月令枚举，时支按十二时辰枚举。
- 性别、顺逆排运和精确起运不参与原局总体。
- 每个合法四柱只出现一次、权重相同。
- 该总体只表示理论结构分布，不代表现实出生频率。

### 生成约束

1. 生成器直接接受四柱，不伪造日期凑出组合。
2. 原局评分必须是四柱纯函数；同一四柱在同一版本下结果相同。
3. 依赖四柱之外数据的规则不得进入理论基准，除非先定义统一、可复算的基准口径。
4. 全集必须得到518,400个唯一键；缺失或重复均为硬失败。
5. 全集只在规则、评分模型或总体版本变化时离线重建。

### 冻结产物

项目不保存518,400张完整命盘，只保存：

- 总体口径、生成器版本、样本数、唯一数和重复数。
- 原始分直方图及SHA-256。
- 分数范围、均值、标准差、分布偏度和关键分位点。
- 候选档与最终档计数。
- 最高、最低同分组的结构类型摘要。
- 引擎、规则、评分模型和基准版本。

## 六、结构分五档与理论审计

单盘使用未舍入结构分保留一位小数直接形成候选档：

| 未舍入结构分 | 候选档位 |
| --- | --- |
| `<= 48` | 偏低 |
| `> 48 且 <= 52` | 中等 |
| `> 52 且 < 75` | 偏高 |
| `>= 75 且 < 78` | 高 |
| `>= 78` | 顶级候选 |

边界来自 518,400 理论合法四柱全集审计并作整数化，不把单盘先换算为百分位再定档。
理论审计仍可保存 `P_low / P_mid / P_high / E / N`，用于检查分布、极值和同分桶，
但页面、专业上下文和 AI 均不显示 P 值。

## 七、高档与顶级资格

裁决顺序为：先按结构分计算候选档，再核高档和顶级资格。

```text
finalIndex = clamp(candidateIndex, 0, ceilingIndex)
```

- 偏低、中等、偏高直接采用结构分候选；病点、清浊、破救和气势已进入六维分，不因同一原因重复降档。
- 高必须明确成格，或属于成而有瑕但已有有效救应；否则最高偏高。
- 顶级候选未通过顶级全部资格时，最高为高。

顶级必须同时满足：

1. 未舍入结构分达到 `78`。
2. 主格严格成立且条件完整。
3. 主作用链全部可达。
4. 清浊达到清或较清。
5. 无未解病点和部分残病。
6. 无仍然生效的直接破格点。
7. 无仍然生效的气势病象。

完全处理过的病点不构成阻断。格名或单一优点不能绕过顶级门槛。

## 八、原局、当前大运与现实边界

### 原局

原局由四柱、规则版本、评分模型和理论基准唯一决定。相同四柱在同一版本下必须返回相同结果。

### 当前大运

当前大运作为第五柱复核合冲刑害、根气、制化、清浊、病药和主格条件，输出阶段状态与当前
发挥档位。第五柱不进入518,400种四柱总体，不生成新的理论P值，也不改变原局P值和原局档位。

### 现实边界

家庭、教育、时代、地区、行业、健康、技能、资金、积累和个人选择没有本模块输入入口。
系统不计算现实承接分，也不把这些因素写入原局分数、理论百分位或当前大运档位；页面只说明
格局层次不等同于现实人生成就排名。

真人反馈只能用于发现缺失条件、误触发或重复计分。任何修正规则必须形成可证伪条件并对全部
命盘统一生效，不能把真人经历作为评分验收答案。


`patternRawScoreBreakdown` 必须包含六维分数、生效权重、加权贡献、组件、规则、证据、模型版本和

当前大运继续由 `evaluateLuckImpact()` 返回 `natalGrade`、`currentGrade`、`activationStatus` 和
`conditionReview`，不改写原局结构分与档位。

### 普通版

- 保留命格、主要格局、格局层次、结构分和白话解释。
- 格局层次只显示档位、主要成败限制和一位小数结构分。
- 不显示权重、规则ID、引擎版本、理论百分位和完整核验链。

### 专业版

- 与普通版保持相同的主格、成败、喜用、原局档位和当前大运结论。
- 显示结构分、六维账本、规则、证据、反证、高档资格和版本，不显示理论百分位。
- 保留专业推理过程，不重复解释基础专业词语的通俗含义。

### AI上下文

AI必须知道五档含义、结构分和资格原因，不得把结构分翻译成财富、学历、社会身份或人生成功比例。
用户另行提供现实条件时可以单独讨论，但不得把这些信息回写评分对象。

## 十、实施顺序

### 阶段一：评分语义与因果去重

1. 统一评分目标为“原局结构可用性”。
2. 为六维账本补齐 `causeId / numericOwner`。
3. 审核潜力与完成度、清浊与病药、承载与气势之间的同源计分。
4. 冻结当前实现作为对照；同一审计轮不得同时修改规则条件和权重参数。

### 阶段二：理论全集

1. 增加纯四柱全集生成器并验证518,400个唯一结构。
2. 分别运行对照模型和候选模型，生成直方图、同分区间、关键分位及直接分档统计。
3. 记录旧百分位分档相对结构分直接分档的诊断迁移。

### 阶段三：极值与边界审计

1. 审查最高、最低全部同分组，不以单张命盘代表极值。
2. 审查结构分 `48 / 52 / 75 / 78` 边界两侧，并保留 P5、P20、P50、P80、P95 作为分布诊断。
3. 覆盖普通已成、严格组合、严格特殊格、成而有瑕、已破、病点全解、部分残病和未解反例。
4. 若发现异常，只修改通用规则、因果归属或单组参数，并完整重建总体。
5. 输出候选权重与维度锚点的敏感性报告，提交用户确认最终数值。

### 阶段四：冻结与接入

1. 用户复核理论成立盘、反例、极值组和边界报告。
2. 用户明确确认后冻结引擎、规则、评分模型、理论基准、直方图和SHA-256。
3. 最后接入结构分、普通版、专业版和AI上下文；同分区间只留在审计报告。

## 十一、验收与版本

### 硬失败

- 全集数量不是518,400或存在重复键。
- 同一四柱在同一版本下得到不同原始分。
- 候选分档误用整数显示分而不是未舍入结构分，导致边界盘跨档。
- 未通过成立规则的结构取得严格格局潜力。
- 同一 `causeId` 在多个维度产生非零影响。
- 已完全处理的病点再次扣分或压档。
- `floorIndex > ceilingIndex` 仍输出档位。
- 顶级结果未通过全部顶级资格。
- 当前大运改写原局结构分或原局档位。
- 页面或AI显示理论百分位，或把结构分表述为现实人口或人生成就排名。
- 未完成全集审计和用户确认即冻结最终权重或维度锚点。

### 必须报告但不自动失败

- 最大同分桶、均值、标准差和分布偏度变化。
- 旧抽样相对全集的P值误差与跨档数量。
- 新旧模型主格、状态、直接候选档和最终档迁移矩阵。
- 最高、最低同分组的结构类型数量。
- 任一维度与总分相关性过高，或大量集中在少数离散值。

### 版本规则

- 格局成立规则变化：升级 `ruleVersion` 和 `engineVersion`，重建理论基准。
- 六维权重、维度锚点、因果归属或公式变化：升级 `scoreModelVersion` 和 `engineVersion`，重建理论基准。
- 理论总体、排序方式、同分政策或直方图变化：升级 `baselineVersion`，重建哈希。
- 只改变页面文字且数据合同不变：不升级评分模型；数据合同变化时升级引擎版本。
